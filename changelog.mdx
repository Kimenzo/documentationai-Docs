## Recent Releases

Review the most recent versions with detailed changelogs.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added support for custom brand colors, including `#3B82F6` as the default blue.
- Introduced `<Video>` and `<Iframe>` components for richer media embeds in your docs.
- Enabled Mermaid diagram rendering directly in code blocks (no JSX wrapper needed).

## Improvements

- Enhanced MDX parsing to better handle nested code blocks with 4 backticks.
- Optimized rendering speed for pages with multiple `<Tabs>` and `<Columns>` components.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- New `<ParamField>` and `<ResponseField>` components for API documentation.
- Added `tags` prop to `<Update>` for filtering changelogs by type.

## Bug Fixes

- Fixed parsing errors from unescaped `{` and `}` in prose text.
- Resolved heading hierarchy issues when skipping from H2 to H4.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## Breaking Changes

- Removed deprecated `<Mermaid>` JSX component; use code blocks only.

## New Features

- Core MDX components: `<Steps>`, `<CodeGroup>`, `<Callout>`, and more.
- Full support for YAML frontmatter with `title` and `description`.

## Initial Setup

- Bootstrap your docs with the default template.

</Update>

## How to Upgrade

Follow these steps to update your Lorenzo Otieno Documentation instance.

<Steps>
  <Step title="Backup Your Docs" icon="database">
    Save your current `docs/` folder to avoid data loss.
  </Step>
  <Step title="Update Package" icon="download">
    Use your preferred package manager.
    <CodeGroup tabs="npm,yarn,pnpm">
```bash
npm install lorenzo-otieno-docs@latest
```

```bash
yarn add lorenzo-otieno-docs@latest
```

```bash
pnpm add lorenzo-otieno-docs@latest
```
    </CodeGroup>
  </Step>
  <Step title="Rebuild and Deploy" icon="rocket">
    Run `npm run build` and deploy to your hosting provider.
  </Step>
</Steps>

<Callout kind="tip">
  Test upgrades in a staging environment first. Check the `<Update>` tags for any breaking changes.
</Callout>
