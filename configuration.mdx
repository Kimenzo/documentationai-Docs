## Brand Colors and Themes

Define your brand colors to match your project's visual identity. Use the primary color `#3B82F6` as a starting point and override CSS variables in your theme file.

<Steps>
  <Step title="Locate Theme File" icon="file">
    Navigate to `docs/theme.config.js`.
  </Step>
  <Step title="Update Colors" icon="edit-3">
    Modify the `colors` object with your hex values.
  </Step>
  <Step title="Apply Changes" icon="refresh-cw">
    Restart your dev server: `npm run dev`.
  </Step>
</Steps>

<CodeGroup tabs="JavaScript,TypeScript">
  ```javascript
  /** @type {import('@lorenzo-otieno/docs').ThemeConfig} */
  const config = {
    colors: {
      primary: '#3B82F6',
      accent: '#1E40AF',
      background: '#FFFFFF'
    }
  };

  export default config;
  ```
  ```typescript
  import type { ThemeConfig } from '@lorenzo-otieno/docs';

  const config: ThemeConfig = {
    colors: {
      primary: '#3B82F6',
      accent: '#1E40AF',
      background: '#FFFFFF'
    }
  };

  export default config;
  ```
</CodeGroup>

<Callout kind="tip">
  Choose colors with good contrast ratios for readability. Test with tools like WebAIM Contrast Checker.
</Callout>

## User Permissions

Manage who can view, edit, or administer your documentation space. Permissions apply at the space and page levels.

| Role          | View Pages | Edit Content | Manage Users | Delete Pages |
|---------------|------------|--------------|--------------|--------------|
| Viewer        | ✅        | ❌           | ❌           | ❌           |
| Editor        | ✅        | ✅           | ❌           | ❌           |
| Admin         | ✅        | ✅           | ✅           | ✅           |

<Tabs>
  <Tab title="Add Editor" icon="user-plus">
    Invite users via email and assign the `editor` role in the dashboard.
  </Tab>
  <Tab title="Restrict Page" icon="lock">
    Set page-specific permissions under **Settings > Access**.
  </Tab>
</Tabs>

## Integration Options

Connect your documentation to external services for seamless workflows.

<ExpandableGroup>
  <Expandable title="GitHub Sync" default-open="true">
    Automatically sync changes from your repo.

    ```bash
    npm install @lorenzo-otieno/docs-github-sync
    ```

    Add to `docs/config.js`:

    ```javascript
    module.exports = {
      integrations: {
        github: {
          repo: 'your-org/your-repo',
          branch: 'main',
          token: process.env.GITHUB_TOKEN
        }
      }
    };
    ```

    <Callout kind="info">
      Generate a GitHub token with `repo` scope for authentication.
    </Callout>
  </Expandable>
  <Expandable title="Slack Notifications">
    Receive alerts for new edits or comments.

    ```javascript
    module.exports = {
      integrations: {
        slack: {
          webhook: 'https://hooks.slack.com/services/YOUR/WEBHOOK'
        }
      }
    };
    ```
  </Expandable>
</ExpandableGroup>
